Chapter 3 – Basic Computer Organization and Design

Instruction Codes, Registers, Timing and Control, Instruction Cycle, Memory-Reference Instructions, I/O & Interrupt, Basic Computer & Accumulator Design
3.1 Overview

Introduction

In this chapter we study a simplified model of a digital computer, often called the basic computer. It is not a real commercial CPU, but it captures all fundamental concepts that you need for exams and for understanding real processors later. We focus on how instructions are encoded, how registers are organized, how control signals are generated in time, and how a simple accumulator-based CPU is designed.

Idea to remember: A computer can be understood as data stored in memory, a set of registers that hold temporary values, and a control unit that repeatedly fetches, decodes, and executes instructions.
3.2 Instruction codes

Instruction Codes

An instruction code is a binary pattern stored in memory that tells the computer which operation to perform and on which data. In the basic computer, each word in memory is 16 bits long, and every instruction is stored in one word.

The most common instruction format for memory-reference instructions is:

Memory-reference format (16 bits):
I | Opcode (3 bits) | Address (12 bits)
Here, I is the addressing-mode bit (0 for direct, 1 for indirect), the next 3 bits are the operation code, and the last 12 bits specify a memory address (allowing 4096 = 212 distinct locations).

Example: If the 3-bit opcode is 010, it may represent the LDA (load accumulator) instruction. If the 12-bit address is 0010 1011 1100, the CPU will load the contents of memory location 0x2BC into the accumulator, either directly or indirectly depending on the I bit.

Types of instructions in the basic computer

For the basic computer model, all instructions are divided into three broad groups by opcode and addressing mode:

  • Memory-reference instructions (e.g., AND, ADD, LDA, STA, BUN, BSA, ISZ): operate on data in memory using the address field.
  • Register-reference instructions: operate directly on the accumulator (AC) and other flags, without an explicit memory operand; usually identified by opcode 111 and addressing bit I = 0.
  • Input–output instructions: transfer data between AC and I/O registers or control interrupts; typically opcode 111 with I = 1.
3.3 Processor and other registers

Computer Registers

The basic computer uses a small set of registers with fixed roles. These registers are connected through a common bus and controlled by the control unit. You should remember their names and functions, because they are frequently asked in exams.

  • AR (Address Register, 12 bits): Holds a memory address for the next read or write operation.
  • PC (Program Counter, 12 bits): Holds the address of the next instruction to be fetched from memory.
  • DR (Data Register, 16 bits): Holds data read from memory, or data to be written into memory.
  • AC (Accumulator, 16 bits): The main processing register used for arithmetic and logic operations.
  • IR (Instruction Register, 16 bits): Holds the current instruction word being decoded and executed.
  • TR (Temporary Register, 16 bits): Used as a temporary storage during certain operations or micro-operations.
  • INPR (Input Register, 8 bits) and OUTR (Output Register, 8 bits): Communicate serially with external input and output devices and in parallel with AC.
  • SC (Sequence Counter, 4 bits): Generates timing steps T0 to T15 for micro-operations.
  • Flip-flops: such as E (extend carry), IEN (interrupt enable), and R (interrupt request).
Important: The basic computer is an accumulator-based machine, meaning that almost every arithmetic or logic instruction uses AC as one of the operands and also stores the result back into AC.
3.4 Computer instructions

Computer Instructions

A computer instruction specifies four things: operation to perform, where operands are located, where to store the result, and how to move to the next instruction. In the basic computer, this information is compressed into the 16-bit instruction word.

Memory-reference instructions use the address field, while register-reference and I/O instructions interpret the address bits as control flags. For example, a register-reference instruction can set or clear specific flip-flops in AC or E depending on which bit is 1 in the instruction.

3.5 Timing and control

Timing and Control

The timing and control unit generates the control signals that tell each register and functional block what to do at each clock pulse. A master clock generator produces uniform clock pulses that drive all flip-flops and registers in the system.

The sequence counter SC is incremented with each clock pulse. Its outputs are decoded to generate timing signals T0, T1, ..., T15. Together with the decoded opcode bits D0 ... D7 (from IR) and current status bits, these timing signals activate the required micro-operations in the correct order.

Example of timing:
At T0, the control unit loads the program counter value into AR and starts a memory read. At T1, the instruction word from memory is loaded into IR and PC is incremented. At T2, the instruction is decoded and the next phase (indirect or direct execution) is selected.
3.6 Instruction execution

Instruction Cycle

The instruction cycle is the repeating sequence of operations that the CPU performs for every instruction in a program. In the basic computer, each instruction cycle is divided into smaller phases, each associated with specific timing signals.

Phases of the instruction cycle

  • Fetch phase: The address in PC is transferred to AR, and the instruction at that address is read from memory into IR. Then PC is incremented to point to the next instruction.
  • Decode phase: The opcode bits of IR are decoded to determine the type of instruction. If the instruction is memory-reference and has indirect addressing, the indirect phase is triggered.
  • Indirect address phase (if needed): If I = 1, the 12-bit address field in IR is used to read the effective address from memory into AR.
  • Execute phase: Depending on the instruction type, one of several execution sequences (memory-reference, register-reference, or I/O) is activated.
The instruction cycle continues endlessly until a HLT (halt) instruction is executed, which stops the sequence counter and disables further micro-operations.
3.7 Memory-reference instructions

Memory-Reference Instructions

Memory-reference instructions use the 12-bit address field of IR to specify an operand in memory. They may use direct addressing (I = 0) or indirect addressing (I = 1). Common memory-reference instructions of the basic computer include:

  • AND: AC ← AC & M[AR] (bitwise AND)
  • ADD: AC ← AC + M[AR] (add memory operand to AC)
  • LDA: AC ← M[AR] (load AC from memory)
  • STA: M[AR] ← AC (store AC to memory)
  • BUN: PC ← AR (unconditional branch)
  • BSA: M[AR] ← PC, PC ← AR + 1 (branch and save return address)
  • ISZ: M[AR] ← M[AR] + 1; if result = 0 then PC ← PC + 1 (used in loops)

For each of these instructions, the control unit generates a specific micro-operation sequence after the fetch and decode phases. For example, LDA will cause a memory read into DR followed by a transfer from DR to AC.

3.8 Input-output and interrupt

Input–Output and Interrupt

The basic computer communicates with external devices using input and output instructions. Two special 8-bit registers are involved: INPR receives serial input from a keyboard or similar device, and OUTR holds data to be sent to an output device such as a printer or display.

Simple I/O instructions include transferring data between AC and these registers and checking flag bits that indicate whether input is ready or output is complete. To avoid wasting CPU time in polling, the basic computer also supports interrupt-initiated I/O using an interrupt enable flip-flop IEN and an interrupt request flip-flop R.

Interrupt cycle (concept)

When an interrupt is enabled and an I/O device sets the request flag, the control unit completes the current instruction, then enters an interrupt cycle. The content of PC is saved (for example, at location 0), PC is loaded with the starting address of the interrupt service routine, and R is cleared. After servicing the device, a branch instruction returns control to the main program.

3.9 Complete basic computer

Complete Computer Description

The complete basic computer includes a 4096 × 16 memory, the processor registers (AR, PC, DR, AC, IR, TR, INPR, OUTR, SC), the adder and logic circuit connected to AC, status flip-flops (E, IEN, R), and the control unit that decodes instructions and generates micro-operations.

Hardware-wise, all registers are connected through a common bus system implemented using multiplexers or tri-state gates. The control unit decides which register places its content on the bus and which register loads from the bus at each timing step. This arrangement minimizes the number of physical wires while still allowing flexible data transfers.

3.10 Design of basic computer

Design of Basic Computer

Designing the basic computer means specifying all its hardware components and the micro-operations they can perform. Major components include the memory unit, the register set, the common bus, the adder and logic circuit, and the control logic (hardwired).

The control logic uses the decoded instruction bits (D0 ... D7), the timing signals (T0 ... T15), and status flags to produce control outputs such as LD, INR, and CLR for each register, as well as select signals for the adder-logic circuit and the bus. A collection of Boolean equations or a control table defines exactly which micro-operations are active at each time step for each instruction.

3.11 Design of accumulator unit

Design of Accumulator Unit

The accumulator unit centers around the 16-bit AC register and an associated adder and logic circuit. AC typically has control inputs LD (load), INR (increment), and CLR (clear). Its input can come from the common bus, from the output of the adder-logic unit, or from DR, depending on the micro-operation being performed.

The adder and logic circuit usually supports operations such as addition, complement, logical AND, and shift operations. For example, instruction CMA (complement AC) will pass the bitwise complement of AC back into AC, while CIL (circular shift left through E) shifts AC left and brings in the E flip-flop as the least significant bit. All these operations are controlled by a small set of select lines from the control unit.

In exam answers, clearly state: “AC is the main processing register of the basic computer. It works with an adder/logic circuit controlled by micro-operations like ADD, AND, complement, and shift, and is connected to memory via DR and the common bus.”
3.12 Practice

Practice Questions (With Answers)

Q1. Define an instruction code. What is the general format of a memory-reference instruction in the basic computer?
Answer: An instruction code is a binary pattern stored in memory that specifies an operation and its operands. In the basic computer, a memory-reference instruction has 16 bits arranged as I | Opcode (3 bits) | Address (12 bits), where I is the addressing-mode bit, the opcode selects the operation, and the address field specifies the memory location of the operand.
Solution approach: Start from the definition (binary representation of an instruction) and then clearly write the 1 + 3 + 12 bit structure, mentioning the purpose of each field.
Q2. List any five processor registers of the basic computer and write one line for the function of each.
Answer: Example set of five registers: (1) AR – holds the address of the memory location to be accessed. (2) PC – holds the address of the next instruction to be fetched. (3) DR – holds data read from memory or data to be written to memory. (4) AC – general-purpose processing register used for arithmetic and logic operations. (5) IR – holds the current instruction being decoded and executed.
Solution approach: Choose any five standard registers (AR, PC, DR, AC, IR, TR, INPR, OUTR) and write a clear, exam-style functional description.
Q3. What is the role of the sequence counter (SC) and the timing signals T0–T15 in the basic computer?
Answer: The sequence counter (SC) is a 4-bit register that is incremented by the clock, and its output is decoded to generate a sequence of timing signals T0–T15. Each timing signal represents a micro-time step in which a specific set of micro-operations is enabled. Together with the decoded opcode and status flags, these timing signals control the exact order in which micro-operations are executed during the instruction cycle.
Solution approach: Explain SC as a small counter, then mention decoding into T0–T15 and state that these signals drive micro-operations in each phase.
Q4. Explain the different phases of the instruction cycle in the basic computer.
Answer: The instruction cycle consists of: (1) Fetch phase – the instruction at the address in PC is fetched into IR, and PC is incremented. (2) Decode phase – the opcode in IR is decoded to determine the instruction type. (3) Indirect address phase – if the addressing bit I is 1, the effective address is obtained from memory and loaded into AR. (4) Execute phase – the control unit generates micro-operations to carry out the instruction, which may be memory-reference, register-reference, or I/O.
Solution approach: Write each phase in order with 1–2 lines: fetch, decode, optional indirect, and execute, emphasizing the role of PC, IR, and AR.
Q5. Write the function of the following memory-reference instructions: AND, ADD, LDA, STA.
Answer: AND: AC ← AC & M[AR]. ADD: AC ← AC + M[AR]. LDA: AC ← M[AR]. STA: M[AR] ← AC. Here M[AR] denotes the contents of the memory location whose address is in AR.
Solution approach: Use symbolic micro-operation notation, always mentioning AC and M[AR]. Keep the format “AC ← expression” or “M[AR] ← AC”.
Q6. Distinguish between register-reference and memory-reference instructions in the basic computer.
Answer: Memory-reference instructions use the 12-bit address field of the instruction to specify a memory operand, may use direct or indirect addressing, and typically perform operations like AND, ADD, LDA, STA, BUN, BSA, ISZ on memory data. Register-reference instructions, identified by opcode 111 with I = 0, do not access memory; instead they interpret the remaining bits of the instruction as control flags to perform operations directly on AC, E, and other internal flip-flops (for example, clear AC, complement AC, shift AC).
Solution approach: Compare on three points: (1) use of address field, (2) type of operand (memory vs. register/flag), and (3) typical examples.
Q7. What is interrupt-initiated I/O? How does the basic computer handle an interrupt request?
Answer: Interrupt-initiated I/O is a technique where an I/O device signals the CPU when it is ready for data transfer, instead of the CPU repeatedly polling the device. In the basic computer, when interrupts are enabled (IEN = 1) and a device sets the request flip-flop R, the CPU completes the current instruction and then enters an interrupt cycle: it saves the current PC value in a fixed memory location (e.g., address 0), loads PC with the starting address of the interrupt service routine, clears R, and executes the service routine. At the end, a branch instruction returns control to the main program.
Solution approach: Define the concept in one line, then describe the steps: finish current instruction, save PC, branch to service routine, clear request, and return.
Q8. Briefly describe the hardware components of the basic computer.
Answer: The basic computer contains: a 4096 × 16 memory unit; a set of processor registers (AR, PC, DR, AC, IR, TR, INPR, OUTR, SC); an adder and logic unit connected to AC; status flip-flops such as E, IEN, and R; a common bus system that interconnects all registers; and a hardwired control unit that decodes instruction codes and generates the timing and control signals required for micro-operations.
Solution approach: List components in a structured way – memory, registers, ALU/adder-logic, bus, status flip-flops, and control unit – each in a few words.
Q9. Explain the design and function of the accumulator (AC) in the basic computer.
Answer: The accumulator AC is a 16-bit general-purpose processing register. It receives data from memory via DR and the common bus, interacts with the adder and logic circuit to perform arithmetic and logic operations, and supplies results back to memory or I/O. AC has control inputs such as LD, INR, and CLR that are driven by the control unit, and many instructions (ADD, AND, LDA, CMA, CIL, CIR) explicitly use AC as their main operand and result register.
Solution approach: Emphasize that AC is central to execution: mention its size, its connection to the adder-logic unit and bus, and the typical micro-operations involving AC.
Q10. Why is a common bus system used in the basic computer? How is it controlled?
Answer: A common bus system is used to reduce the number of physical wires needed to interconnect all registers and the memory. Instead of having dedicated lines between every pair of registers, a shared set of lines carries data from one selected source register to one or more destination registers. The control unit enables the appropriate source on the bus and asserts the load (LD) signals of the destination registers at the correct timing signals, usually using multiplexer select inputs or tri-state enable lines.
Solution approach: First explain the purpose (hardware economy and flexibility), then state that the control unit selects the data source and destination through control signals synchronized with the clock.
